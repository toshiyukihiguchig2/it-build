# 業務システム資材

各業務システムの TRUNK 資材をローカルに配置するためのフォルダです。

## フォルダ構成

```
systems/
├── {システム名}/
│   ├── README.md   ← コミット対象（概要・SVNパス・主要ファイル情報）
│   └── src/        ← コミット対象外（SVNからコピーした実資材）
```

## 資材の同期手順

### SVN からコピーする場合
```bash
# systems/{システム名}/src/ に TRUNK の内容をエクスポート
svn export svn://[SVNサーバ]/[リポジトリ]/[システム名]/trunk systems/[システム名]/src --force
```

### SVN 作業コピーからコピーする場合（チェックアウト済みの場合）
```bash
# Windows の場合
xcopy /E /Y "[SVN作業コピーのパス]\*" "systems\[システム名]\src\"
```

### 更新（差分のみ取得）したい場合
```bash
svn update [SVN作業コピーのパス]
xcopy /E /Y "[SVN作業コピーのパス]\*" "systems\[システム名]\src\"
```

## 注意事項

- `src/` フォルダは `.gitignore` によりコミットされません（ローカル専用）
- GitHub Copilot がコードを参照できるのは **ローカルに配置されている間のみ** です
- 依頼対象のシステムが確定したら、上記手順で資材を配置してから `/support` を使用してください
- `README.md` は各システムの概要・SVNパスを記録しているため、コミット対象です
