# Review チーム サンプル依頼集

`/support` 起動後、レビュー依頼として使える例です。

---

## コードレビュー部

```
以下のコードをレビューしてください。
言語: Java
観点: バグ・セキュリティ・可読性

[コードをここに貼る]
```

```
このPRのレビューコメント案を作ってください。
変更内容: ユーザー認証処理のリファクタリング

[変更コードをここに貼る]
```

```
以下のコードにSQLインジェクションのリスクがないか確認してください。

[コードをここに貼る]
```

## 設計レビュー部

```
以下の基本設計書をレビューしてください。
要件定義書と突き合わせて、抜け漏れ・矛盾を指摘してください。

[設計書をここに貼る]
```

```
テーブル設計をレビューしてください。
正規化・インデックス・命名規則の観点で確認してください。

[ER図またはDDLをここに貼る]
```

## ドキュメントレビュー部

```

観点: わかりやすさ・抜け漏れ・曖昧表現

[手順書をここに貼る]
```

```
障害報告書のレビューをしてください。
第三者が読んで原因・対策が伝わるか確認してください。

[報告書をここに貼る]
```
