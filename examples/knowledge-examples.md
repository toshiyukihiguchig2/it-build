# Knowledge チーム サンプル依頼集

`/support` 起動後、ナレッジ管理として使える例です。

---

## ナレッジ整理部

```
社内のWikiが散らかっていて情報が見つかりません。
以下のページ一覧を整理して、カテゴリ分けと統廃合の提案をしてください。

[ページ一覧をここに貼る]
```

```
障害対応のナレッジが担当者のメモに散在しています。
以下のメモ群を整理して、Wiki に登録できる形にまとめてください。

[メモをここに貼る]
```

---

## 標準化・手順書部

```
毎月行っているサーバのパッチ適用作業を手順書にしたいです。
以下の作業メモをもとに、誰でも実行できる手順書を作ってください。

[作業メモをここに貼る]
```

```
新人に引き継ぐための定期作業チェックリストを作りたいです。
以下の作業一覧をチェックリスト形式に整理してください。

[作業一覧をここに貼る]
```

```
以下の手順書が属人的で他の人が実行できません。
改善してください。判断基準が曖昧な箇所を明文化してください。

[手順書をここに貼る]
```

```
[問い合わせ一覧をここに貼る]
```

```
「VPNが繋がらない」という問い合わせが月10件来ています。
ユーザーが自己解決できるFAQ記事を作成してください。
```

---

## テンプレート管理部

```
障害報告書のテンプレートを作ってください。
必須項目: 発生日時・影響範囲・原因・対処内容・再発防止策
提出先: 部長・関係システム担当者
```

```
システム変更申請書のテンプレートを作ってください。
承認フロー: 申請者 → IT部門長 → 業務部門長
必須項目: 変更概要・影響範囲・リスク・ロールバック手順・作業日時
```

```
議事録テンプレートを作ってください。
想定会議: 定例MTG / 障害振り返り / プロジェクト会議
```
