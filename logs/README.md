# ログ配置フォルダ

本番サーバから取得したログをここに配置します。
このフォルダ配下のログは `.gitignore` によりコミットされません（ローカル専用）。

## フォルダ命名規則

```
logs/{YYYYMMDD}-{システム名}/
├── {ログ種別}.log   # アプリログ・バッチログ・OSログ等
└── memo.md          # 取得時の状況メモ（任意）
```

**例：**
```
logs/20260329-受注システム/
├── app.log
├── batch.log
└── memo.md
```

## ログ取得・配置手順

### サーバからローカルへのコピー（SCP）
```bash
scp user@[サーバ]:/var/log/[システム]/app.log logs/[YYYYMMDD]-[システム名]/
```

### 複数ファイルまとめてコピー
```bash
scp -r user@[サーバ]:/var/log/[システム]/ logs/[YYYYMMDD]-[システム名]/
```

### Windows サーバの場合（共有フォルダ経由）
```
\\[サーバ]\logs\ からコピーして logs\[YYYYMMDD]-[システム名]\ に配置
```

## 配置後の使い方

ログを配置したら `/support` を起動して以下のように依頼する：

```
logs/[YYYYMMDD]-[システム名]/ にログを配置しました。
[システム名] で [エラー内容 / 事象] が発生しています。
原因調査をお願いします。
```

## 注意事項

- 個人情報・機密情報が含まれるログは取り扱いに注意すること
- 調査完了後はローカルのログを削除すること（本番ログの残留防止）
- 事例として残す場合は `incidents/` に記録する（生ログではなく要約で）
