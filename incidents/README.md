# 過去事例（インシデント記録）

解消した障害・調査事例を記録するフォルダです。
次回同様の事象が発生したとき、Copilot がこのフォルダを参照して過去事例を活用します。

## ファイル命名規則

```
incidents/{YYYYMMDD}-{システム名}-{事象の概要}.md
```

**例：**
```
incidents/20260301-受注システム-夜間バッチORA-01555.md
incidents/20260315-在庫システム-API接続タイムアウト.md
```

## 記録のタイミング

- 障害対応が完了したら **当日中** に記録する
- 軽微なものでも「同じ事象が再発しそう」と思ったら記録する
- 原因が特定できなかった未解決事例も記録しておく（`## 状態: 未解決` と明記）

## テンプレート

`_template.md` をコピーして作成すること。
